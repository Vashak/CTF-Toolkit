# 🗡️ CTF Crypto Toolkit

This repository contains my personal collection of Python exploits and scripts developed to solve Cryptography and Hardware Reverse Engineering challenges during Capture The Flag (CTF) competitions.

The toolkit covers a variety of attack vectors, from breaking classic algorithms (RSA, AES) to exploiting hardware vulnerabilities (LFSR) and side-channel data.

> **🌟 Highlight:** Looking for something more advanced? Check out my standalone project where I bypassed ChaCha20 padding using linear cryptanalysis on GF(2): [https://github.com/TuoUsername/ChaCha20-Linear-Exploit]

## 🧰 The Arsenal

Below is an index of the available scripts and the vulnerabilities they exploit:

### 🎲 Pseudo-Random Number Generators (PRNG)
* **`SRNG_CLONE.py`**: Performs a *State Recovery* attack by cloning the internal state of Python's Mersenne Twister. Predicts future numbers generated by the server.

### 🔐 Block Ciphers
* **`AES_CTR_ATTACK.py`**: Exploits Nonce length manipulation to trigger a counter extension, bypassing server crashes to extract the Keystream (*Keystream Reuse Attack*).

### 📐 Asymmetric Cryptography
* **`RSA_GCD_ATTACK.py`**: Exploits the reuse of prime numbers (*Common Factor Attack*) on poorly generated RSA moduli by calculating the Greatest Common Divisor (GCD) to extract the private key.

### ⚡ Side-Channels & Oracles
* **`SIDE_CHANNEL_ATTACK.py`**: Measures server response times (latency) to guess the hidden flag character by character.
* **`ELF_ATTACK.py`**: Exploits the standard error output (stderr) of a compiled Linux binary to build an error-based oracle.

### ⚙️ Hardware Reverse Engineering
* **`LOGIC_GATE_ATTACK.py`**: Brute-force attack against the physical architecture of an LFSR (Linear Feedback Shift Register). Tests all 63 tap configurations and initial seeds, filtering for readable ASCII output.
* **`REVERSE_XOR.py`**: Exploits the involutory nature of XOR to collapse a complex stream cipher algorithm into a single known plaintext byte (*Known Plaintext Attack*).
